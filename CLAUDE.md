# CLAUDE.md · Junior_IT

Память проекта для Claude CLI. Читается в начале каждой сессии. Держим коротко и
по делу — глубину выносим в `meta/`.

## TL;DR

**Junior_IT** — бесплатный курс программирования для подростков **11–14 лет** при ЮДО
(Всеволожский район, Ленобласть). Ведёт Димас — senior-разработчик
(`ASM C C++ Python JS Pascal Dart C# Java`), [@DmitrTRC](https://github.com/DmitrTRC).

Материалы — статические **HTML/CSS/JS** + Markdown. Деплой — **GitHub Pages**
на каждый push в `main` → https://dmitrtrc.github.io/Junior_IT/

Формат: вебинар 75 мин через MAX (запись в Ecamm Live). Дети **смотрят на уроке,
делают дома** по шпаргалке и стартеру.

## ⛔ Жёсткие запреты (не нарушать никогда)

Это материалы для детей и публичный репозиторий с записями, которые живут годами.

**В материалах и в кадре:**
- НЕ использовать символику СССР, ФСБ, силовых ведомств, советских органов, любые
  политически нагруженные принты/знаки.
- НЕ писать слово **«дружина»** (взрослые → ассоциации с СССР, дети не понимают).

**В коммуникации с родителями:**
- НЕ употреблять «контроль», «проверки», «инспекции».
- НЕ использовать «дружина» / «отряд» в рассылках.
- НЕ предлагать «кружковую методику как в школе».

**В работе с кодом и правками:**
- НЕ вываливать слишком много за раз — Димас сам остановит и попросит развёрнуто.
- НЕ навязывать альтернативы выбранному стеку (HTML→CSS→JS, vanilla, без фреймворков)
  без явного запроса.
- Правки — **хирургические**: не переписывать окружающий контекст ради одной строки.

## 🎓 Методология (почему так, а не иначе)

- Старт с **HTML+CSS**, не Python/C: нулевой барьер, мгновенный визуальный результат,
  девочки не отваливаются, естественный мост к JS.
- На вебинаре дети **только смотрят и слушают**, НЕ повторяют параллельно.
- Теория — не дольше **2 минут** подряд. Каждые 3–5 мин — пинг в чат / вопрос / реакция.
- CSS — кульминация первого урока, не приложение к HTML.
- В первых 3–4 уроках НЕ всплывают: «семантика», «DOCTYPE», «валидация», «парсинг», «DOM».
- На каждый урок — **резервный финальный файл** (`index-final.html`) на случай, если
  живой кодинг сломается.
- Демки — в трёх эстетиках (геймер / спорт / арт), чтобы каждый ребёнок узнал себя.
- 75 минут — потолок (дальше внимание мертво). Можем сократить до 60.

**Главный урок из практики:** `getElementById` оказался стеной (слишком много магии
сразу — объект, метод, свойство, цепочка через точку). Поэтому между «оживляем сайт»
и «события/анимация» вставлен мост — **урок 2.5 «Сам язык: консоль-калькулятор»**
(чистый JS в консоли, без DOM). Подробности — `meta/lesson-01-postmortem.md`.

## 🎨 Бренд-кит

Не выдумывать цвета/шрифты — брать отсюда. Полная версия в `meta/project-instructions.md`.

**Палитра (CSS-переменные):**

| Token | HEX | Назначение |
|---|---|---|
| `--bg` | `#0a0a14` | глубокий тёмный фон |
| `--card` | `#1a1a2e` | основной тёмный (карточки) |
| `--card-2` | `#16213e` | вторичный тёмный |
| `--deep` | `#0f3460` | акцентный тёмный |
| `--yellow` | `#ffd60a` | подпись бренда |
| `--orange` | `#ff5722` | эстетика «спорт» |
| `--green` | `#00e676` | live / success |
| `--cyan` | `#00ffc8` | эстетика «геймер» |
| `--magenta` | `#ff0096` | эстетика «геймер» |
| `--bg-light` | `#faf7f2` | светлый фон для печати |

**Консоль (подсветка кода):** в консольных блоках уроков — Dracula-палитра:
`#8be9fd` (kw), `#f1fa8c` (str), `#ffb86c` (num), `#ff79c6` (bool), `#6272a4` (com),
`#7fd0ff` (вывод), фоны `#15151c`/`#0c0c11`. Это функциональный слой, не декор —
в токены не входит, но единообразен по всей линейке. Не «чинить» в отдельном уроке.

**Шрифты:** Headings — Space Grotesk 700 (Bebas Neue для спорта, Fraunces для арта);
Body — Manrope 400/500/600; Code/labels — JetBrains Mono 400/700.
⚠️ Bebas Neue и Fraunces на Google Fonts — **без кириллицы**. В демках с русскими
заголовками (`demos/sport.html`, `demos/art.html`) — намеренные Cyrillic-safe фолбэки
`Arial Black` / `Georgia`. Не свопить обратно на Bebas/Fraunces — «Саня»/«Лиза» уйдут в tofu.

**Принципы дизайна:**
- Градиенты, мягкие тени, плавные hover, скруглённые карточки.
- Минимум комментариев в коде (только для неочевидного).
- Никаких хрупких `position:absolute` декораций в hero — ломаются на широких экранах.
- Кастомные SVG-баннеры, не генераторы вроде capsule-render.
- В Markdown — эмодзи-заголовки, бейджи через `img.shields.io`.

> Сайты курса — это самореклама курса. Если первая страница выглядит как презентация
> из 2003 — ребёнок закроет вкладку и не вернётся.

## 🧩 Анатомия урока

Каждый урок — папка `lessons/lesson-NN-тема/` с фиксированным набором файлов
(эталон — `lesson-01-html-css/`):

| Файл | Что это |
|---|---|
| `README.md` | навигация по уроку + «что ребёнок умеет к концу» |
| `scenario.md` | сценарий с таймингом по минутам (+ чек-лист готовности в конце) |
| `live-code.md` | пошаговый живой кодинг (~8 шагов) |
| `glossary.md` | словарик терминов простыми словами |
| `cheatsheet.html` | шпаргалка для детей (отправляется в чат в КОНЦЕ урока) |
| `homework.html` | домашка-стартер (тоже в конце урока) |
| `slides.html` | слайды для шаринга (переключение ← / →) |
| `index-final.html` | резервный финальный код |
| `demos/gamer.html` | демка геймер (тёмная, неон, `--cyan`/`--magenta`) |
| `demos/sport.html` | демка спорт (яркая, `--orange`) |
| `demos/art.html` | демка арт (тёплая, элегантная) |

Шпаргалку и домашку отправляем **в конце** урока, не до.

## 🛠 Slash-команды (`.claude/commands/`)

| Команда | Назначение |
|---|---|
| `/new-lesson NN slug "тема"` | развернуть новый урок по анатомии и бренду |
| `/deploy ["сообщение"]` | ревью изменений → commit → push в main (триггерит GH Pages) |
| `/postmortem NN` | разбор урока + выводы в `meta/lesson-NN-postmortem.md` |
| `/brand-check [путь]` | аудит материалов на бренд + запреты |
| `/parent-msg "тема"` | черновик сообщения родителям по правилам коммуникации |

Сабагент `brand-guardian` — ревьюер материалов (бренд + запреты), вызывается через Task.

## ✅ Рабочие конвенции

- Язык материалов и общения — **русский**. С детьми — на «ты».
- Имена папок уроков: `lesson-NN-короткий-slug` (NN с ведущим нулём).
- Перед `/deploy` — всегда показывать diff и резюме, не пушить молча.
- Новые/переписанные материалы прогонять через `/brand-check` (или brand-guardian)
  до коммита.
- `students/` и `graphify-out/` — приватное/генерируемое, не коммитим.
- Глубокий контекст: `meta/project-instructions.md` (полные инструкции проекта) и
  `meta/lesson-01-postmortem.md` (что сработало, что нет, открытые вопросы).
