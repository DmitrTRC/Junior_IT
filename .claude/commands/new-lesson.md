Создай новый урок Junior_IT.

- Номер урока (NN, с ведущим нулём): **$1**
- Slug папки: **$2**
- Тема: **$ARGUMENTS** (всё после slug — это формулировка темы)

## Что сделать

1. Создай папку `lessons/lesson-$1-$2/` и заполни полный набор файлов по анатомии
   урока из `CLAUDE.md`. Эталон структуры и тона — `lessons/lesson-01-html-css/`
   (прочитай его файлы как образец прежде чем писать):
   - `README.md` — навигация + блок «что ребёнок умеет к концу урока»
   - `scenario.md` — сценарий с таймингом по минутам, в конце чек-лист готовности
   - `live-code.md` — пошаговый живой кодинг (~8 шагов)
   - `glossary.md` — словарик терминов простыми словами
   - `cheatsheet.html` — шпаргалка для детей
   - `homework.html` — домашка-стартер
   - `slides.html` — слайды (переключение ← / →)
   - `index-final.html` — резервный финальный код
   - `demos/gamer.html`, `demos/sport.html`, `demos/art.html` — три эстетики

2. Жёстко соблюдай методологию и запреты из `CLAUDE.md`: теория ≤2 мин блоками,
   дети смотрят (не повторяют), без «семантика/DOCTYPE/DOM» на ранних уроках,
   три эстетики демок, потолок 75 мин.

3. HTML — по бренд-киту (палитра-токены, шрифты, hover, без хрупких
   `position:absolute` в hero). Не выдумывай цвета/шрифты.

4. После генерации прогони материалы через сабагент `brand-guardian`
   (или `/brand-check lessons/lesson-$1-$2`) и почини найденное.

5. Не пуши. Покажи дерево созданных файлов и краткое резюме — деплой отдельно
   через `/deploy`.

Сначала уточни у Димаса одной фразой, какой стек/фокус у урока, если из темы
неоднозначно. Не вываливай весь объём без подтверждения направления.
