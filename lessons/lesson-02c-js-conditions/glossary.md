# 📖 Словарик урока 2.6 — «Программа решает: если…то»

> Объясняем простыми словами, без «функция/событие/DOM/булево/оператор сравнения». Эти слова — позже. Сегодня только: сравнить и выбрать.

| Слово | Простыми словами | Пример |
|---|---|---|
| **сравнить** | Спросить у браузера «больше? меньше? равно?». В ответ — `true` или `false`. | `10 > 5` |
| **`true` / `false`** | Ответ «да» / «нет». Новый вид значения: не число и не текст. | `7 === 7` → `true` |
| **`>` `<` `>=` `<=`** | Больше, меньше, больше-или-равно, меньше-или-равно. | `ball >= 50` |
| **`===`** | «Равно ли?». Три знака равно — это **сравнить** (вопрос), ответ `true`/`false`. | `ochki === 100` |
| **`!==`** | «Не равно ли?». | `otvet !== "да"` |
| **`=`** | Один знак равно — это **положить** в коробочку. Это НЕ вопрос. | `let ochki = 0` |
| **`if (...)`** | «Если…». Сделает то, что в фигурных скобках, **только если** в круглых `true`. | `if (ball >= 50) { ... }` |
| **`{ }` фигурные скобки** | «Сюда — что делать, когда условие сработало». | `{ alert("Зачёт") }` |
| **`else`** | «Иначе». Что делать, если условие `false`. | `else { alert("Мимо") }` |
| **`else if`** | «А иначе если…». Ещё одна ветка выбора между `if` и `else`. | `else if (xp >= 500) { ... }` |
| **`&&`** | «И» — оба условия должны быть `true`. | `age >= 11 && age <= 14` |
| **`||`** | «Или» — хватит одного `true`. | `день === "сб" \|\| день === "вс"` |
| **условие** | То, что стоит в круглых скобках `if`. Это всегда вопрос с ответом да/нет. | `(ochki >= 50)` |

---

## Главная мысль урока одной строкой

Раньше программа шла **по прямой** и всегда делала одно и то же. С `if` она **выбирает** дорогу: сравнила — и решила, что делать.

## Две ловушки, из-за которых «не работает»

1. **`=` вместо `===`.** Один `=` кладёт значение (`ochki = 50` — просто поставит 50!), а сравнивает — три: `ochki === 50`. Внутри `if` почти всегда нужно `===`.
2. **Сравнил текст вместо числа.** Ответ `prompt` — текст. `"100" > "9"` даст `false` (сравнило по буквам, а не по величине). Перед сравнением оберни в `Number(...)`.
