# Урок 2.6 — Программа решает: если…то

**Тема:** Условия в консоли. Сравнение (`>`, `<`, `===`), `if` / `else` / `else if`. Программа выбирает, что делать. Без файлов и без DOM.
**Длительность:** 60–75 минут
**Возраст:** 11–14 лет
**Место в курсе:** продолжение урока 2.5 (консоль-калькулятор) — последний шаг «чистого языка» перед возвратом кнопок на уроке 3.

## Зачем этот урок

На уроке 2.5 программа шла **по прямой**: спросить → посчитать → показать. Всегда одно и то же. Сегодня учим её **решать**: «если баллов хватило — зачёт, иначе — ещё чуть-чуть». Это первый момент, когда программа делает не одно и то же, а зависит от ответа. Всё так же в консоли, без единой кнопки. На уроке 3 этот выбор повесим на клик — и игры/реакции станут естественными.

## Что ребёнок умеет к концу урока

- Сравнивает значения: `>`, `<`, `>=`, `<=`, `===`, `!==` — и понимает, что в ответе `true` или `false`
- Различает `=` (положить в коробочку) и `===` (сравнить) — и знает, почему их путают
- Пишет `if (условие) { ... }` — сделать что-то, только если ответ «да»
- Добавляет `else` — «иначе» — и `else if` — ещё одну ветку выбора
- Собирает программу «спроси → сравни → реши» через `prompt` + `Number` + `if`
- Помнит ловушку: сравнивать ответ `prompt` без `Number` — текст, и `"100" > "9"` соврёт
- Может дома собрать свой мини-тест/калькулятор-решатель по шпаргалке

## Связь с соседними уроками

- **Назад (урок 2.5):** «помните мини-программу спросить → посчитать → показать? Сегодня она научится выбирать ответ».
- **Вперёд (урок 3):** этот же `if` повесим на клик кнопки — и получится реакция/мини-игра.

## Методическая заметка

Демки запускают решающую мини-программу через `prompt`/`alert` сами при открытии (для «вау» на вебинаре). Это всё ещё чистый язык — никакого `getElementById`. Дома дети повторяют те же строки в консоли по шпаргалке. Главная грабля урока — `=` вместо `===`; вынесена в шпаргалку и в чек-лист «не работает».
