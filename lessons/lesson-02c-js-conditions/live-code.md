# 💻 LIVE-CODE: «Программа решает — если…то» (урок 2.6)

**Стартовая точка:** НИКАКИХ файлов. Открываешь браузер → консоль (F12 → Console). Всё пишем прямо там.
**Зачем этот урок:** на прошлом уроке программа шла по прямой — спросить, посчитать, показать. Сегодня научим её **выбирать**: сравнила — и решила, что делать. Всё ещё чистый язык в консоли, без страницы и кнопок. Кнопки вернём на следующем уроке — и `if` сразу ляжет на клик.
**Принцип:** одна строка = один ответ браузера. Между шагами — пинг в чат. Теории не дольше 2 минут подряд.

---

## ШАГ 0 · Напоминаем, где мы

🎬 Открываешь вкладку → `F12` → вкладка **Console**. Курсор после `>`. Печатаешь `2 + 2` → `4`.

🎤 «В прошлый раз мы собрали программу: спросить → посчитать → показать. Но она всегда делает одно и то же. А настоящая программа умеет РЕШАТЬ: если так — одно, иначе — другое. Сегодня про это. И снова нам хватит одной консоли.»

💬 **В чат:** «Кто помнит прошлую программу про возраст? Поставьте 🙂. Сейчас она научится выбирать ответ.»

> ⚠️ Снова покажи, КУДА печатать (вкладка Console). Дома путают с Elements — главный «не работает».

---

## ШАГ 1 · Сравнение: браузер отвечает «да» или «нет»

🎤 «Браузер умеет не только считать, но и сравнивать. Спросим у него.»

```js
10 > 5
3 > 8
7 === 7
```

🎬 Ответы: `true`, `false`, `true`.

🎤 «`true` — это "да", `false` — "нет". Новый вид ответа: не число и не текст, а да/нет. Заметьте: "равно?" — это ТРИ знака равно `===`, не один.»

💬 **В чат:** «Кидайте сравнение — посчитаю: больше у вас рост или у соседа по парте?»

> 💡 Возьми пару чисел из чата и сравни вживую. `>=` — «больше или равно», `<=` — «меньше или равно».

---

## ШАГ 2 · `=` положить, `===` сравнить (главная ловушка)

🎤 «Вот тут все спотыкаются. Один знак равно и три знака равно — это РАЗНОЕ.»

```js
let ochki = 5
ochki === 5
ochki === 10
```

🎬 Ответы: коробочка завелась, потом `true`, потом `false`.

🎤 «Один `=` — "положи 5 в коробочку". Три `===` — "а равно ли?", и браузер отвечает да/нет. Если внутри условия написать один `=` — ты не спросишь, а ПЕРЕЗАПИШЕШЬ. Это грабля номер один.»

💬 **В чат:** «🤯 кто думал, что = и === одно и то же?»

> ⚠️ Проговори медленно. `=` кладёт, `===` спрашивает. Внутри `if` почти всегда нужно `===`.

---

## ШАГ 3 · `if` — сделать, только если «да»

🎤 «Теперь главное слово урока: `if` — "если". Сделает то, что в фигурных скобках, только когда в круглых `true`.»

```js
let ball = 80
if (ball >= 50) {
  alert("Зачёт!")
}
```

🎬 `80 >= 50` — это `true`, поэтому выскакивает «Зачёт!».

🎤 «В круглых скобках — вопрос. В фигурных — что делать, если ответ "да". Поменяю балл на 30 — и ничего не выскочит: условие `false`, браузер просто пропустит.»

```js
ball = 30
if (ball >= 50) {
  alert("Зачёт!")
}
```

💬 **В чат:** «Какой порог поставили бы вы? 50? 60?»

> ⚠️ Круглые скобки — для условия, фигурные — для действия. Пропустишь фигурные — частая ошибка дома.

---

## ШАГ 4 · `else` — а иначе

🎤 «Сейчас, если балла не хватило, не происходит ничего. Добавим "иначе" — `else`.»

```js
let ball = 30
if (ball >= 50) {
  alert("Зачёт! 🎉")
} else {
  alert("Почти — нужно 50")
}
```

🎬 Балл 30 → выскакивает «Почти — нужно 50».

🎤 «`if` — одна дорога, `else` — вторая. Программа всегда пойдёт ровно по одной. Это и есть выбор.»

💬 **В чат:** «🔥 если поняли, как программа выбирает дорогу.»

---

## ШАГ 5 · Спроси и реши (кульминация)

🎤 «Соберём настоящую решающую программу. Спросим балл — и сама решит, зачёт или нет.»

```js
let ball = Number(prompt("Сколько баллов набрал?"))
if (ball >= 50) {
  alert("Зачёт! 🎉")
} else {
  alert("Почти. Нужно 50, у тебя " + ball)
}
```

🎬 Вводишь `72` → «Зачёт!». Перезапусти, введи `40` → «Почти. Нужно 50, у тебя 40».

🎤 «Видите `Number`? Он обязателен. `prompt` отдаёт ТЕКСТ, а текст сравнивается по буквам — `"100" > "9"` даст `false`, хотя 100 больше. `Number` превращает в настоящее число.»

💬 **В чат:** «🔥 у кого получилось бы сделать свой тест на зачёт?»

> ⚠️ Самая коварная пара граблей вместе: забыл `Number` (сравнил текст) ИЛИ написал `=` вместо `===`. Если «ведёт себя странно» — проверь эти две вещи.

---

## ШАГ 6 · Несколько веток: `else if`

🎤 «Дорог может быть больше двух. Между `if` и `else` вставляем `else if` — "а иначе если".»

```js
let xp = Number(prompt("Сколько у тебя XP?"))
if (xp >= 1000) {
  alert("🥇 Золото")
} else if (xp >= 500) {
  alert("🥈 Серебро")
} else {
  alert("🥉 Бронза — качайся дальше")
}
```

🎬 `1500` → золото, `700` → серебро, `100` → бронза.

🎤 «Браузер проверяет сверху вниз и идёт по ПЕРВОЙ подошедшей ветке. Зашёл в золото — серебро и бронзу даже не смотрит.»

💬 **В чат:** «Какие ранги придумали бы вы для своей игры?»

---

## ШАГ 7 · (если есть время / для сильных) «И» и «ИЛИ»

🎤 «Можно проверить сразу два условия. `&&` — "и" (оба верны), `||` — "или" (хватит одного).»

```js
let age = Number(prompt("Сколько тебе лет?"))
if (age >= 11 && age <= 14) {
  alert("Ты как раз наш Junior!")
} else {
  alert("Курс для 11–14, но заходи :)")
}
```

🎤 «`age >= 11 && age <= 14` — между 11 и 14 включительно. Дома попробуйте `||`: суббота ИЛИ воскресенье — выходной.»

> 💡 Скучающему («я знаю if») — задача: «спроси оценку 1–5 и выведи словом: 5 — отлично, 4 — хорошо, 3 — норм, иначе — подтянись».

## ИТОГ на экране (что прозвучало)

- Сравнение даёт `true` / `false` (да/нет): `>` `<` `>=` `<=` `===` `!==`
- `=` положить в коробочку · `===` сравнить (внутри `if` — три равно!)
- `if (условие) { ... }` — сделать, только если `true`
- `else` — иначе · `else if` — ещё ветка
- `prompt` + `Number` + `if` = программа решает сама
- Ловушка: сравнивать текст без `Number` (`"100" > "9"` → `false`)

## 🆘 Если живой кодинг встал

1. **Ничего не происходит** → внутри `if` написано `=` вместо `===`? Один `=` не спрашивает, а кладёт.
2. **Сравнение «врёт»** → забыл `Number(prompt(...))`? Текст сравнивается по буквам.
3. **«Красная ошибка»** → чаще кавычки-ёлочки, русская раскладка или забытая `{` / `}`. Прочитай слово после `Error`.
4. **Совсем сломалось** → открой `index-final.html`, дальше веди по нему: там готовые строки с ответами.
