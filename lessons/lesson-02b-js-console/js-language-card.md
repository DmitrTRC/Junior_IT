# 🧭 JS — паспорт языка (памятка ведущего)

> Шпаргалка **для себя**, не для детей. Чтобы при переключении ASM/C/C++/C#/Java/Python/Pascal → JS мозг быстро встал на рельсы. Только то, на чём полиглот реально спотыкается.

## Паспорт за 10 строк

| Свойство | JS | Чем удивляет |
|---|---|---|
| **Парадигма** | мульти: императив + функциональный (функции — значения) + прототипная ООП | `class` — сахар над прототипами, не как C++/Java |
| **Типизация** | динамическая + **слабая** (неявные приведения) | тип у значения, не у переменной |
| **Числа** | один тип `number` = IEEE-754 double | **нет** int/float/long; `0.1+0.2 ≠ 0.3` |
| **Case** | **строго** регистрозависимый | `Score` ≠ `score`; конвенция — `camelCase` |
| **Объявление** | `const` / `let` (блочные), `var` — легаси | по умолчанию `const`, нужно менять — `let` |
| **Равенство** | `===` строгое (без приведения) | `==` с приведением — **не использовать** |
| **`;`** | опц. (ASI), но ставь | ASI режет `return` на новой строке |
| **Пустые `[]` `{}`** | **truthy** | в Python пустые — falsy! |
| **`this`** | по **месту вызова**, не определения | стрелки `=>` берут `this` лексически |
| **Среда** | JIT, без `main()`, ES-модули | файл = скрипт; модули strict by default |

## 8 типов (всё остальное — object)

`number` · `bigint` · `string` · `boolean` · `null` · `undefined` · `symbol` · `object`
(функция и массив — это `object`). Проверка: `typeof x`. **Грабля:** `typeof null === "object"` (историческая бага), `typeof []  === "object"` (массив → `Array.isArray()`).

## Falsy — выучить наизусть

`false` · `0` · `-0` · `0n` · `""` · `null` · `undefined` · `NaN`

Всё прочее — truthy. В т.ч. `"0"`, `"false"`, `[]`, `{}`, `" "`. `NaN !== NaN` (проверка — `Number.isNaN`).

## Грабли при переключении с …

| Пришёл из | Что НЕ так в JS |
|---|---|
| **C / C++** | нет типов в объявлении; `==` коварен (бери `===`); нет указателей/перегрузки операторов; объекты — по ссылке |
| **C# / Java** | `this` динамический (зависит от вызова!); нет настоящих классов под капотом; нет перегрузки операторов; `int`/`double` слиты в `number` |
| **Python** | блоки — `{}`, не отступы; `camelCase`, не `snake_case`; пустые `[]`/`{}` **истинны**; `null` **и** `undefined`; `===` вместо `==`; объявляй через `let`/`const` |
| **Pascal / Basic** | `=` это присваивание, сравнение — `===`; `{}` блоки; индексация с **0**; массивы динамические и разнотипные |

## Микро-гочи (то, что укусит на уроке)

```js
"5" + 5            // "55"  — + со строкой склеивает (число→строка)
"5" - 5            // 0     — а - приводит к числу
0.1 + 0.2          // 0.30000000000000004
10 / 3             // 3.333…  целочисленного деления нет → Math.trunc(10/3) // 3
let a = 4          // объявление возвращает undefined (видно в консоли)
typeof NaN         // "number"
[] == ![]          // true  — классика приведений; живи на ===
```

## Шпаргалка инструментов

- **Строки:** UTF-16, неизменяемы. Шаблоны: `` `${имя}, тебе ${age}` ``. Кавычки `'` = `"`.
- **Объявления:** `const` нельзя переприсвоить (но объект внутри мутируется). `let` — блочная область + TDZ. `var` забыть.
- **Функции:** первоклассные. `function f(){}` (hoisted, свой `this`) vs `const f = () => {}` (лексический `this`, не hoisted). Примитивы — по значению, объекты — по ссылке.
- **Безопасный доступ:** `obj?.prop` (optional chaining), `a ?? b` (nullish: только для `null`/`undefined`, не для `0`/`""`).
- **Перебор:** `for (const x of arr)` — значения; `for (const k in obj)` — **ключи** (не путать!). Массивы: `map/filter/reduce/forEach`.
- **null vs undefined:** `undefined` = не присвоено/нет; `null` = намеренно пусто (ставишь сам).
- **Строгий режим:** в модулях и `class` — автоматически; иначе `"use strict"` сверху.

> Якорь на каждый день: **числа — все double, сравнение — `===`, `this` — по вызову, пустые коллекции — истинны, регистр — важен.** Этого хватает, чтобы не путаться вслух перед детьми.
