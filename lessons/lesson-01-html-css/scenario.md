# 🎬 СЦЕНАРИЙ ВЕБИНАРА: «Первый сайт за 75 минут»

**Аудитория:** дети 11–14 лет, разный уровень, формат — вебинар (Zoom / Google Meet / любой).  
**Длительность:** 75 минут.  
**Тема:** делаем свою личную страничку на HTML + CSS.

---

## 🎒 Что подготовить ЗА ДЕНЬ до вебинара

### На твоём компьютере
- [ ] **VS Code** открыт, тема контрастная (например, "Default Dark+"), шрифт **18–20pt минимум**. В зуме мелкий шрифт не читается.
- [ ] Открыты в браузере во вкладках (готовы переключиться):
  - `01_slide_intro.html` — стартовый слайд
  - `demo_gamer.html` — пример «геймер»
  - `demo_sport.html` — пример «спорт»
  - `demo_art.html` — пример «арт»
  - `cheatsheet.html` — шпаргалка
- [ ] Пустая папка `Рабочий стол/site` для живого кодинга
- [ ] Браузер с одной вкладкой `about:blank` для показа результата
- [ ] Окно с шарингом экрана настроено: **VS Code слева, браузер справа** (либо на двух мониторах, либо разделить экран). Так дети сразу видят: написал → результат.

### Что отправить ученикам ЗА ДЕНЬ
В письме / чате родителям:
1. Ссылку на вебинар.
2. **Просьба не делать параллельно** — «сегодня смотрим и слушаем, делать будем дома сами, шпаргалку дам».
3. Ничего устанавливать заранее НЕ нужно — это снизит порог входа и количество "у меня не работает" в чате.

### Чат вебинара
- [ ] Назначить **второго ведущего/модератора** если есть — следить за чатом, отвечать «привет», подсвечивать вопросы. Если нет — приучить себя смотреть в чат каждые 2–3 минуты.

### 🟡 [0:10 – 0:15] Минимум теории (ровно 3 концепции)

🎬 Переключаешься на `02_slide_html_basics.html` (или просто открываешь блокнот/VS Code с тремя пунктами крупно).

🎤 «Всего три вещи нужно знать, чтобы начать:»

**Концепция 1.** «Сайт — это просто текстовый файл. Один файл с расширением `.html`. Никакой магии. Браузер открывает этот файл и превращает его в красоту.»

**Концепция 2.** «Внутри файла мы пишем **теги**. Тег — это команда браузеру, что показать. Выглядит так:»

```html
<тег>содержимое</тег>
```

🎤 «Открыли — `<тег>`. Положили внутрь что хотим показать. Закрыли — `</тег>` со слэшем. Как бутерброд: хлеб сверху, начинка, хлеб снизу.»

**Концепция 3.** «У тегов есть разные имена. Каждое имя — это своя команда. `h1` — большой заголовок. `p` — абзац текста. `img` — картинка. И так далее. Сейчас покажу.»

🎤 «ВСЁ. Это вся теория. Теперь пишем.»

> ⚠️ **Не говори** слова: «семантика», «DOCTYPE», «валидация», «парсинг», «DOM». Они придут на 4–5 урок.

### 🟡 [0:35 – 0:55] CSS — делаем красиво

🎬 Возвращаешься к коду. На экране пока **уродская** страничка с чёрным текстом на белом фоне.

🎤 «Посмотрите честно: страничка работает, но выглядит как привет из 1995. Сейчас починим. Знакомьтесь — CSS. Это язык, который говорит браузеру: "сделай красиво".»

🎤 «CSS пишется внутри специального тега `<style>`. Я добавлю его в самом верху.»

**Добавляешь в самое начало файла:**

```html
<style>
  body {
    background: #1a1a2e;
    color: white;
    font-family: Arial, sans-serif;
    padding: 40px;
  }
</style>
```

🎬 Обновляешь — страница превращается из уродины в норму. **Это момент "вау".**

🎤 «Видели? Одно изменение — и совсем другой сайт. Разбираем что я написал:»

- `body` — это вся страница целиком.
- `background` — цвет фона.
- `color` — цвет текста.
- `font-family` — шрифт.
- `padding` — отступ внутри (чтобы не прилипало к краям).

🎤 «Цвета можно писать словами — `red`, `blue`, `yellow`. Или кодом — решётка и шесть символов. Это называется HEX. Где брать — расскажу через минуту.»

**Добавляешь стиль для заголовка:**

```html
<style>
  body {
    background: #1a1a2e;
    color: white;
    font-family: Arial, sans-serif;
    padding: 40px;
  }
  h1 {
    color: #ffd60a;
    font-size: 60px;
    text-align: center;
  }
</style>
```

🎬 Обновил. Заголовок стал жёлтым, огромным, по центру.

🎤 «Видите принцип? Имя тега → фигурные скобки → внутри свойства.»

💬 **Вопрос в чат:** «Какой цвет хотите для заголовка? Напишите название по-английски: red, blue, green, pink, что хотите.»

> Зачитываешь 2-3 ответа: «Маша — pink, отлично. Дима — orange, тоже супер.»

**Делаешь картинку красивой:**

```html
img {
  width: 300px;
  border-radius: 20px;
  border: 4px solid #ffd60a;
}
```

🎬 Обновил. У картинки теперь рамка и закруглённые углы.

🎤 «`border-radius` — закругление углов. Поставите 50% — получится круг. Попробуйте дома.»

**Финальный аккорд — hover-эффект на ссылку:**

```html
a {
  color: #ffd60a;
  text-decoration: none;
  background: #16213e;
  padding: 10px 20px;
  border-radius: 8px;
  display: inline-block;
  transition: 0.3s;
}
a:hover {
  background: #ffd60a;
  color: #1a1a2e;
  transform: scale(1.1);
}
```

🎬 Наводишь мышкой на ссылку — она становится жёлтой и увеличивается.

🎤 «`:hover` — это магия. Делает действие, когда мышка наводится. Так выглядят почти все красивые кнопки в интернете.»

💬 **Вопрос в чат:** «Зацените, у кого получилось понять как работает hover? Кто хочет такое же — поставьте 🔥 в чат.»

## 🆘 ЕСЛИ ЧТО-ТО ПОШЛО НЕ ТАК

### Тишина в чате первые 10 минут
Не паникуй. Назови по именам 2-3 человек из списка участников: «Маша, ты тут? Что любишь?». Если совсем мертво — открой опрос/poll Zoom: «Какая игра нравится больше?» с вариантами. Тапнуть проще, чем писать.

### Кто-то пишет «непонятно»
Не объясняй заново. Скажи: «Запомни этот момент. Запиши вопрос. После урока разберём отдельно — или родители помогут по шпаргалке». Иначе застрянешь на одном ребёнке и потеряешь остальных.

### Кто-то пишет «уже всё знаю, скучно»
«Отлично. Сегодня тогда покажи мне в чат — какой бы ты градиент сделал для своей страницы? Накидай комбинацию из 3 цветов.» Дай задачу повыше, а не споришь.

### Вылетел из звонка / технические проблемы
Заранее запись урока должна вестись с момента старта. Если что — возобновляешь, говоришь «продолжаем, кто опоздал — посмотрит запись».

### Технические проблемы у ученика
В чат: «Не парься, сегодня смотрим. Дома сделаешь сам. Шпаргалка покроет.»
